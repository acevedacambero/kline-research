# K 线结构概率研究台使用说明书

适用网站：[https://skyland.us.ci/](https://skyland.us.ci/)

## 一、系统用途

本系统是一套股票研究工具，用于查看沪深股票日 K 线、审计历史标签、分析结构特征、扫描高分股票、验证因子和研究组合表现。

它不是证券交易软件，不会自动买卖股票，也不应被当作投资建议。

## 二、重要概念

### 1. 信号日

信号日是你想研究的历史日期。例如选择 `2025-01-02`，表示假设在该日收盘后发现这只股票，研究其后是否存在可执行买入机会以及未来表现。

系统不会使用信号日之后的数据计算当天特征，以避免未来数据泄漏。

### 2. P1 标签

P1 用于回答“后来发生了什么”，包括：

- 样本是否具备研究资格；
- T+1 至 T+3 是否可以买入；
- 实际入场日期和价格；
- 未来 5、10、20、60 个交易日收益；
- 路径触发结果；
- 最大回撤；
- 标签成熟日期；
- 停牌、涨停、数据不足等异常。

### 3. P2 特征

P2 描述信号日当天的股票状态，分为趋势、价格位置、动量、量价关系和交易行为五类。它只描述“当时是什么样”，不保证未来上涨。

### 4. P3 结构评分

P3 将 P2 特征转换为 0 至 100 分的结构评分：

- 80 至 100：结构较强；
- 60 至 79：偏强；
- 40 至 59：中性；
- 20 至 39：偏弱；
- 0 至 19：结构较差。

注意：80 分不等于 80% 的上涨概率。

### 5. 成熟标签

只有未来观察期完整结束，标签才算成熟。例如 P60 需要观察未来 60 个有效交易日。尚未成熟的样本不会作为正式结果使用。

### 6. 前复权

正式计算统一使用前复权价格，避免分红、送股和拆股造成价格跳变。

## 三、首次进入系统

### 第一步：打开网站

访问 [https://skyland.us.ci/](https://skyland.us.ci/)，按页面提示完成登录。

### 第二步：检查顶部状态

正常情况下应显示：

- 本地服务正常；
- 数据源上线 Gate：通过；
- 研究运行 Gate：模型就绪；
- 标签数据兼容：全部一致；
- P3 评分数据：4859/4859；
- 行情覆盖率接近 100%。

如果长期显示“正在连接”，等待数秒后刷新页面。

### 第三步：区分按钮用途

普通用户常用：

- 扫描 P6 高分样本；
- 计算并审计；
- 验证 P4 单因子；
- 运行 P5 概率校准；
- 训练 P7 基线模型；
- 训练 P7 多特征模型；
- 运行 P7 Walk-forward；
- 验证 P8 高分组合。

数据维护按钮：

- 拉取代表样本；
- 高速下载全市场；
- 补全短历史；
- 生成 P1 标签；
- 生成 P2 特征；
- 生成 P3 评分；
- 执行数据源上线 Gate。

顶部已经显示数据和模型就绪时，日常研究不需要重新执行数据维护任务。

## 四、数据覆盖与自动维护

### 全市场覆盖台账

1. 打开“数据状态”中的“全市场覆盖台账”。
2. 首次使用点击“重新检查覆盖”。结束后会显示证券总数、已有缓存、可研究数量和待修复数量。
3. 使用“查看状态”筛选缺少缓存、日线历史不足、行情过期、行情缺口、文件不可读或近似复权证券。
4. 点击“修复异常证券”，系统会把异常证券加入后台修复队列。任务结束后再次运行覆盖检查，确认覆盖率变化。

### 日常增量更新

- 需要立即更新时，点击“更新今日行情”。系统只拉取最近窗口，并与已有历史合并。
- 希望系统自动维护时，在“自动维护与备份”中点击“开启自动更新”。默认在工作日 18:30 执行；若当时已有重任务，本次会安全跳过并显示原因。

### 备份

点击“立即备份并校验”。只有状态变成完成，且备份数量增加，才表示备份已经通过完整性校验。网页不提供直接恢复按钮，避免误覆盖运行中的数据库；恢复由管理员停止服务后执行。

## 五、单只股票研究

1. 在“P1 标签审计台”选择上海或深圳。上海股票代码通常以 6 开头，深圳通常以 0 或 3 开头。
2. 输入股票代码或名称关键词，并从候选列表中选择股票。
3. 选择历史信号日。初次使用建议点击“选择最近 P60 成熟日”，系统会自动选择同时具备至少 250 日历史、且未来 P60 观察窗口已经结束的有效交易日。
4. 点击“计算并审计”。

系统会同时读取日 K 线、P1 标签、P2 特征、P3 评分、交易规则和前复权状态。
结果加载后可点击“导出当前审计报告 JSON”，将证券、信号日、行情覆盖区间、P1/P2/P3 结果及版本依赖保存到本地，供复核和归档。
成功审计后，地址栏会保存交易所、证券代码和信号日；可收藏该地址，或复制给其他已获授权的用户打开复核。

## 六、查看 K 线图

图表包括：

- 日 K 线；
- 成交量；
- MA5、MA10、MA20、MA60；
- 信号日；
- 实际入场日；
- 计划卖出日；
- 路径触发点；
- 最大回撤峰值和谷值。

均线可以简单理解为：MA5 代表短线，MA10 代表短中期，MA20 约为一个交易月，MA60 约为一个季度。

常见观察方法：

- 股价在多条均线上方，通常代表趋势偏强；
- MA5、MA10、MA20 由上到下排列，通常属于多头结构；
- 股价远离 MA20 或 MA60，可能意味着短期过热；
- 放量上涨和缩量回调通常比无量上涨更健康；
- 跌破均线不代表必然下跌，需要结合其他特征。

## 七、查看 P1 标签

### 1. 样本资格

不合格的常见原因：历史不足 250 个交易日、字段缺失、前复权异常、新股无涨跌幅限制、未来数据不足或无法得到可执行入场。

### 2. 可执行入场

系统从信号日后的第一个交易日开始检查 T+1 至 T+3，可能返回正常入场、顺延入场、放弃、停牌、涨停不可买、无涨跌幅限制或数据不足。

实际研究应优先查看可执行口径，而不是仅看理论收益。

### 3. P5/P10/P20/P60 收益

分别表示入场后约 5、10、20、60 个交易日的结果。页面可能同时显示绝对收益、可执行收益、基准超额收益和顺延卖出收益。

### 4. 路径结果

路径标签会检查持有过程，而不只看最终收益。同一天同时触发上涨和下跌阈值时，系统保守判定为失败。

### 5. 最大回撤

最大回撤表示持有过程中从阶段高点到后续低点的最大跌幅。即使最终盈利，也应检查中间是否承受过大损失。

### 6. 标签成熟日期

只有当前日期晚于成熟日期，结果才可用于正式历史验证。

## 八、查看 P2 特征

### 趋势

观察是否站上 MA20、MA60，均线是否多头排列，以及中期趋势方向。

### 位置

观察价格在近期高低区间中的位置、距离近期高点的幅度和偏离均线的程度。

### 动量

观察近期收益、涨跌速度、动量持续性以及是否过热或快速转弱。

### 量价

观察成交量是否放大、上涨是否得到成交量支持，以及是否出现放量下跌。

### 交易行为

观察振幅、涨跌停状态、异常行为以及历史特殊状态是否采用近似推断。

## 九、P6 高分扫描

1. 找到“P6 高分扫描”。
2. 选择全部、上海或深圳市场。
3. 初学者建议将最低分设为 70。
4. 截至日期留空时使用最新可用日期，也可以指定历史日期复盘。
5. 点击“扫描 P6 高分样本”。
6. 从结果中选择股票，带入 P1 审计台逐只检查。
7. 如需在 Excel 中整理，可以导出 CSV。

扫描结果只是候选列表，不应直接作为买入清单。

## 十、P4 单因子验证

P4 用于判断分数越高时，历史结果是否总体更好。

操作步骤：

1. 选择收益口径，初学者可先选择 P20 计划收盘卖出。
2. 点击“验证 P4 单因子”。
3. 查看分桶收益、胜率、秩相关、95% 置信区间和独立时段数量。

理想情况是低分组表现较弱、高分组表现较强。秩相关大于 0 表示评分方向与结果总体一致；接近 0 表示关系很弱；小于 0 表示评分方向可能相反。

## 十一、P5 概率校准

P5 用于检查 P3 分数能否解释为上涨概率。

1. 选择收益口径。
2. 分桶数建议使用 10。
3. 点击“运行 P5 概率校准”。

主要指标：

- Brier Score：越低越好；
- Log Loss：越低越好；
- ECE：越接近 0 越好，高于 0.10 通常需要谨慎。

当前 P3 分数用于排序，不能直接解释为概率。

## 十二、P7 模型

### 1. 基线模型

点击“训练 P7 基线模型”。它只使用 P3 分数预测未来收益是否为正。重点查看训练和测试样本数、测试准确率、ROC AUC、模型系数及状态。

### 2. 多特征模型

点击“训练 P7 多特征模型”。它综合使用 P3 分数、多头排列、20 日收益、量比和 20 日波动率。重点查看样本外 AUC、准确率和特征权重。

### 3. Walk-forward

点击“运行 P7 Walk-forward”。系统会按时间顺序使用早期数据训练、后期数据测试，并在多个窗口中重复验证。

重点查看每个窗口的 AUC、准确率和方向是否一致。只在个别窗口有效的模型不够稳定。

### 4. 模型注册表

系统会保存多特征模型、基线模型和 Walk-forward 结果，并记录数据版本、标签版本、训练截止时间和模型状态。

## 十三、P8 组合验证

P8 用于研究历史上选择评分最高的一部分股票时，组合表现如何。

1. 选取比例建议先使用 10%。
2. 选择收益口径。
3. 截至日期可以留空。
4. 成本默认 10 bps，滑点默认 5 bps。
5. 点击“验证 P8 高分组合”。

重点查看：净组合收益、全样本收益、净超额收益、胜率、最大回撤、Sharpe、Calmar 和净值曲线。

P8 是日线研究模拟，不等于真实成交回测。

## 十四、研究实验历史

每次运行 P4、P5、P6、P7 或 P8 后，系统都会自动保存完整实验，不需要手工点击保存。

1. 打开“研究实验历史”，按实验类型筛选记录。
2. 查看运行时间、主要结果、数据快照和代码版本。
3. 点击“重新载入”，旧实验结果会恢复到对应的 P4–P8 结果面板。
4. 需要判断参数调整是否有效时，在“对比两个同类实验”中选择两条相同类型的记录，再点击“生成对比”。

数据快照哈希不同表示两次实验使用的基础行情清单不同；此时指标变化不一定只来自参数调整，应结合运行时间和证券数量解释。

## 十五、新手推荐流程

1. 检查顶部数据状态。
2. 在 P6 扫描 70 分以上股票。
3. 选择一只股票带入 P1 审计。
4. 查看 K 线和均线。
5. 查看 P2 五类特征。
6. 阅读 P3 评分原因。
7. 检查 P1 可执行入场和回撤。
8. 查看 P4 历史统计支持。
9. 查看 P5 概率可靠性。
10. 查看 P7 样本外稳定性。
11. 查看 P8 组合层面的超额收益。
12. 综合形成研究结论，不要只依据一个分数。

## 十六、常见错误

### 页面显示“正在连接”

等待 5 至 10 秒后刷新页面，并确认网址为 `https://skyland.us.ci/`。

### 请求失败 500

刷新页面，检查交易所和股票代码，或换一个历史日期。避免在全市场任务运行时启动另一个重任务。

### 请求失败 502、503 或 524

可能是服务重启、服务器繁忙或代理超时。等待约 30 秒后刷新，不要连续重复点击。

### 显示“数据不足”

可能是新股历史不足、信号日太早、信号日太接近今天、P60 尚未成熟、停牌或价格字段异常。

### 显示“近似规则”

说明历史 ST 或特殊状态无法完整还原。样本仍可研究，但应降低结论可信度。

### 按钮不可点击

通常表示系统正在运行其他任务，或数据、特征尚未加载完成。等待状态更新即可。

## 十七、普通用户不要频繁执行的操作

- 高速下载全市场；
- 补全短历史；
- 生成 P1 标签；
- 生成 P2 特征；
- 生成 P3 评分；
- 重试 P1 错误。

这些是数据维护任务，可能持续较长时间并占用服务器资源。

## 十八、可信度原则

1. 相关性不等于因果关系。
2. 历史有效不代表未来有效。
3. 高分不等于高概率。
4. 胜率高不代表收益风险比好。
5. 平均收益可能由少数极端样本造成。
6. 应同时查看收益、回撤、置信区间和样本外结果。
7. 标记为 `review` 的结果只能继续研究，不应直接用于交易。
8. 系统不模拟完整实盘成交环境。
9. 不要使用借款、杠杆或生活必需资金验证研究结论。

## 十九、系统边界

目前支持上海和深圳股票、日线、前复权、历史标签、结构特征、评分、扫描、概率校准、模型验证和组合研究。

目前不支持北交所、实时行情、分钟级策略、自动下单、券商账户连接、实盘持仓、完整税费模型、冲击成本、融资融券、期权和期货。

## 二十、最简操作口诀

> 先看状态，再做扫描；  
> 选出候选，逐只审计；  
> 分数只作排序，不当概率；  
> 既看收益，也看回撤；  
> 既看历史，也看样本外；  
> 标记 review，只研究不交易。
新增的“跨时期稳定性”会把样本按时间分成三个区段。如果三个区段方向一致，显示“方向稳定”；否则显示“需要复核”。p 值是单段原始显著性，q 值是多重检验校正后的结果，优先查看 q 值，不能只挑最漂亮的一个时期。

P7 默认在训练截止日和测试集之间留出七个自然日隔离区，并剔除训练截止日尚未成熟的标签。该规则用于降低相邻样本和未来标签造成的数据泄漏；实验记录会保存被 purge、embargo 和未成熟过滤的样本数量。

## 二十一、检查特征漂移

完成 P2 特征和 P3 评分后，点击顶部“检查特征漂移”，或进入“特征与评分漂移监控”区域点击“开始检查”。普通用户可保留默认的近期 60 个、基准 250 个交易日。

- “稳定”：近期数据分布和历史基准接近，可以继续常规研究。
- “需要观察”：已有明显变化，应结合分市场结果复核，暂缓把旧模型结论直接用于近期样本。
- “发现漂移”：至少一项指标变化超过警戒线，应重新验证特征、评分和模型，不要只凭旧实验结论。
- “数据不足”：本地 P2/P3 数据未覆盖两个窗口，先完成数据、特征和评分生成。

PSI 衡量整体分布变化，标准化均值偏移衡量平均水平变化，缺失率变化用于发现近期数据质量退化。页面同时显示上海和深圳的状态，可帮助判断变化是全市场现象还是单一市场异常。每次检查都会自动保存到“实验历史”，以后可以选择两个同类记录进行比较。

## 二十二、选择当前模型

P7 模型训练通过后会出现在“模型注册表”。状态为“训练完成”的候选模型可以点击“设为当前”；启用后显示绿色“当前模型”和“使用中”。同类型模型任一时刻只有一个当前版本，再选择旧模型即可回切。系统会拒绝启用需要复核或依赖旧版数据的模型。

“当前模型”表示后续研究功能默认引用的受控版本，不代表模型已经达到实盘标准，也不会触发自动交易。
