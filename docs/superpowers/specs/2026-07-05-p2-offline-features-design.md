# P2 离线特征层设计

## 目标与边界

P2 在现有 AkShare 本地数据底座和 P1 标签层之上，生成可复现、无未来数据泄漏的日频研究特征。首期只做特征计算、版本化落盘、批量任务和单票审计；不做特征评分、因子筛选、概率校准、模型训练或交易信号。

## 特征范围

每个证券、每个有效交易日生成一行特征，包含五组：

1. 趋势：MA5/10/20/60、均线多空排列、均线斜率、收盘价相对均线偏离。
2. 位置：20/60/120/250 日区间位置、距同期最高价回撤。
3. 动量：5/10/20/60/120 日收益。
4. 量价：量比、成交额、成交量历史分位、收益波动率、日内振幅。
5. 交易行为：涨停次数、连续一字板、跳空、停牌间隔。

窗口未满时字段返回空值，并保留 `availableHistory`，不使用未来数据填补。

## 价格口径

- 交易规则特征使用原始行情：涨停、跌停、一字板和跳空均以当日可观察原始价格及当日规则判断。
- 趋势、位置和形态特征使用本地派生前复权行情，保证横截面和图形口径一致。
- 研究收益和波动率使用本地派生总回报序列，避免分红送转形成伪收益。
- 禁止把不同提供方返回的复权序列拼接；所有派生序列必须来自同一份原始快照和对应因子。

## 架构与数据流

新增 `kline.features` 模块，分为纯计算内核与批处理存储两层。计算内核接收同一证券按日期升序排列的 raw、qfq、total-return 数据，输出按日特征表；批处理层遍历本地证券快照，隔离单票失败并写入版本化 Parquet。

特征数据集身份由以下四项共同确定：

- `snapshotVersion`
- `factorVersion`
- `tradingRuleVersion`
- `featureDefinitionVersion`（首版 `daily-features-v1`）

输出目录和清单记录上述依赖、生成时间、证券数、行数、失败数及错误明细。任一依赖版本变化时，不复用旧结果。

## API 与界面

新增异步特征构建接口，沿用现有任务 ID 和轮询机制；重复启动时阻止并发构建。新增单票特征审计接口，按交易所、代码、日期返回特征值、数据口径、窗口可用性和版本信息。

页面在现有研究台增加“生成 P2 特征”入口、任务进度，以及单票 P2 审计区。审计区按五组展示，不在前端重新计算金融指标；后端返回的空值显示为“历史不足”。

## 异常与质量

- 单票原始、复权或总回报序列缺失：该证券失败并进入任务错误清单，不中断全量任务。
- 单日字段异常：受影响特征为空，并在该行 `reasons[]` 中记录原因。
- 历史 ST 近似状态继续携带 `isApprox`、`reason` 和规则版本，涨停类特征可审计。
- 质量报告至少统计证券数、特征行数、各字段缺失率、近似规则行占比和失败证券。

## 测试与验收

纯计算测试覆盖窗口边界、均线与斜率、区间位置、动量、量比/分位、波动率/振幅、涨停与一字板、跳空、停牌间隔，以及严格的信号日截断。批处理测试覆盖版本依赖、断点复用和单票失败隔离；API 测试覆盖任务与审计契约；前端测试覆盖任务轮询、分组展示和空值说明。

验收标准：全量现有缓存可生成版本化特征集；任取证券和日期可解释每项特征的价格口径与窗口；新增测试与原 P0/P1 测试全部通过。
