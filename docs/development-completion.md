# 功能开发完成度台账

更新日期：2026-07-18。功能闭环、全量研究层重建、P4–P8 正式实验、模型比较和生产部署均已执行；当前只保留恢复/回滚演练与受 Cloudflare Access 保护的浏览器端最终人工验收。

## 1. 数据血缘与失效机制：已完成

- 行情快照贯穿 P1、P2、P3、实验记录和模型依赖。
- 状态接口报告当前、过期、缺失、孤儿、重复和不可读产物。
- 审计、扫描、训练、漂移和组合验证拒绝读取过期产物。
- 行情更新后旧研究产物和旧快照模型不会计入研究验收。

## 2. 一键研究流水线：已完成

- “一键日常更新与研究维护”依次执行行情增量更新、质量台账、P1、P2、P3。
- 支持仅变更证券和全量研究层两种模式。
- 未变化证券跳过下游重建；旧产物未补齐的证券仍自动进入重建集合。
- 五阶段统一进度、单票错误隔离、DuckDB 单写者、持久化状态、服务重启恢复和安全取消均已实现。
- P4–P8 不在流水线中自动运行。

## 3. 数据下载与修复：已完成

- 多源降级、有限重试、超时、错误分类、失败重试队列和历史补全已实现。
- 增量数据按日期合并，短窗口不能覆盖完整历史。
- 普通股票、指数、错误交易所和北交所生产数据已隔离。
- 行情、P1、P2、P3 孤儿及已有替代品的旧产物可扫描、隔离或确认删除。

## 4. P4–P8 研究工作台：已完成

- 所有研究入口检查当前数据版本，过期时阻断并说明原因。
- 实验记录支持筛选、比较、恢复参数和按恢复参数重新运行。
- 模型注册、启用、历史切换和数据快照依赖可审计。
- P6 优先使用可推理且依赖当前数据快照的受控当前模型；否则明确退回 P3 排序。
- P8 展示成本、滑点、基准、净超额、风险指标、平均单向换手率和可执行卖出口径。

## 5. 页面操作流程：已完成

- 页面按数据维护、P1–P3、单票审计、P6、P4/P5、P7、P8、实验历史和研究 Gate 组织。
- 数量由接口动态返回；任务、错误、过期、近似和历史不足状态统一展示。
- 长表格和图表具备响应式溢出保护，主要操作完成后给出结果或下一步提示。

## 6. 运维管理：功能已完成，演练待最终验收

- 页面汇总磁盘、数据新鲜度、任务失败和 provider Gate 告警。
- 定时任务会跳过正在运行的重任务。
- VPS 临时备份可校验、下载到本地并在哈希确认后删除；大型备份提供自动拉取脚本。
- 部署、回滚、备份和停止服务后的数据库恢复脚本已提供。
- 维护配置、provider 探测、备份、产物清理和模型启用保留不含密钥的持久化管理审计记录。
- 实际恢复演练、线上回滚演练和容量阈值确认留到最终验收。

## 7. 文档与契约：已完成并持续同步

- README、小白使用说明、OpenAPI 和 TypeScript 类型与当前功能同步。
- 当前生产范围只包含沪深日线研究；北交所、实时行情、自动交易、周月线正式标签和逐笔撮合不在范围内。

## 统一验收执行结果

- 沪深 5,204 只证券的 P1→P2→P3 已完成 15,612/15,612 个阶段单元，错误 0。
- P4、P5、P6、P7 基线、P7 多特征、P7 walk-forward 和 P8 七类正式实验均已登记，当前快照下缺失和过期实验均为 0。
- P7 基线样本外 AUC 为 0.4584，多特征 AUC 为 0.4722，均未达到启用条件；系统正确拒绝提升弱模型，P6 继续明确回退 P3 排序。
- provider 上线 Gate 已重新通过：腾讯股票/指数、Sina 原始行情/复权因子和交易日历全部成功；东方财富诊断失败不影响生产路由。
- 数据新鲜覆盖率 99.98%，不可读证券和身份错配均为 0；1 只已退市证券保留旧末日行情，不低于 95% 生产新鲜度门槛。
- 后端 319 项、前端 41 项测试、Ruff 与前端生产构建通过；生产健康检查通过。
- 研究 readiness 的刷新、审计、建模检查全部通过。最终验收仍显示“尚未选择当前模型”，这是弱模型未达门槛后的预期安全状态，不应通过强制启用消除。
